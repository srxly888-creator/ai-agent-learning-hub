## 🎯 解决的痛点

### **API 能用，但太原始**
- 命令行复杂
- 需要记住各种参数
- 输出格式不友好

### **Portainer 这类又偏重**
- 需要部署 Web 面板
- 资源占用大
- 配置复杂

### **很多时候你只是想**
- 看 repo
- 查 tag
- 删几个镜像
- **根本不想开一整套 Web 面板**

---

## ✨ crtui 的优势

### **1. 终端原生**
- ✅ 在终端里完成所有操作
- ✅ 无需 Web 面板
- ✅ 轻量级

### **2. 智能读取凭据**
- ✅ 自动读取 Docker/Podman 凭据
- ✅ 无需重复配置
- ✅ 安全便捷

### **3. 丰富的信息展示**
- ✅ tag 细节
- ✅ 镜像元数据
- ✅ layer 大小
- ✅ 多架构清单

### **4. 实用功能**
- ✅ 列出仓库
- ✅ 查询 tag
- ✅ 删除镜像
- ✅ 复制 docker pull 命令

### **安装**

```bash
# 待补充具体安装命令
```

### **基本使用**
```bash
# 列出仓库
crtui repos

# 查询 tag
crtui tags <repo>

# 删除镜像
crtui rmi <repo>:<tag>

# 复制 pull 命令
crtui copy <repo>:<tag>
```

## 🎯 与其他工具的对比

### **vs Docker CLI**
- **Docker CLI**: 命令复杂，输出不友好

### **vs Portainer**
- **Portainer**: 需要 Web 面板，资源占用大
- **crtui**: 终端原生，轻量级

### **vs Harbor**
- **Harbor**: 企业级，部署复杂
- **crtui**: 个人/小团队，开箱即用
