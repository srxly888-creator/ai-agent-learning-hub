## Part 1：Codex 基础入门

### 00:00 开场介绍

Codex 是 OpenAI 的全新**超级应用**，一个**统一的 AI 代理工具**。它可以：

- ✅ 创建和编辑设计稿
- ✅ 做研究、生成文档
- ✅ 创建和部署完整的 Web 应用
- ✅ 创建动态图形发布视频
- ✅ 创建可导出到 Canva 的高质量融资 PPT
- ✅ 甚至可以用 Swift 创建高质量的 iOS 应用

**Codex 把编码、协作、浏览器和计算机操控（Computer Use）全部整合到了一个界面里**。

视频分为两部分：
- **Part 1**：基础功能介绍
- **Part 2**：多任务实战，同时做多个项目

### 02:54 下载 Codex

前往 OpenAI 官网下载 Codex 桌面应用。下载安装后即可使用。

> 💡 **前提条件**：你需要一个 OpenAI 账号。

### 03:20 界面总览

打开 Codex 后，左侧面板有 **5 个选项**：

1. **Search（搜索）**
2. **Chats（聊天列表）**
3. **Plugins（插件/技能）**
4. **Automations（自动化）**
5. **Projects（项目）**

界面整体类似于 ChatGPT 的聊天界面，但功能远比 ChatGPT 强大——它是一个能**控制你整个电脑**的 AI 代理。

### 03:56 聊天、提示词与内置搜索

- Codex 有**内置网页搜索**功能，就像 ChatGPT 一样，可以自动搜索互联网
- 你可以直接输入问题，比如搜索 OpenAI 桌面应用的最新功能
- 它会自动搜索并给出回答

> 📌 **小贴士**：你不需要手动指定搜索，AI 代理足够聪明，会自动判断是否需要搜索。

### 04:53 创建项目

**项目（Project）是 Codex 中最核心的概念之一。**

#### 创建步骤：
1. 点击左上角的 **新建项目** 按钮
2. Codex 会打开系统的文件夹选择器（Finder）
3. 创建一个新文件夹（比如 `Codex Projects`），作为你的工作空间
4. 选择该文件夹并点击"打开"

#### 为什么要创建项目？
- 给 AI 代理一个**具体的工作目录**
- 所有在该项目下创建的文件都会保存在这个文件夹中
- 不同项目的文件互不干扰，井然有序

#### 实操演示：
Riley 创建了一个名为 `Codex Desktop Research` 的项目文件夹，用于存放所有关于 Codex 桌面应用的研究文件。

### 07:37 创建电子表格

你可以让 AI 代理直接帮你创建电子表格！

#### 操作步骤：
1. 在项目聊天中输入："请把我刚告诉你的新功能整理成一个表格"
2. AI 会自动创建一个 `.xlsx` 文件
3. 文件保存在项目文件夹的 `outputs` 目录中
4. 点击文件旁的 **Open** 按钮可以在侧边栏预览
5. 可以点击**全屏按钮**进入全屏查看模式

#### 编辑表格：
- 在全屏模式下，你可以继续用自然语言编辑，比如：
  - "请删除'来源页面'这一列"
  - AI 会直接修改文件

> 💡 **关键点**：所有代理创建的文件都会自动保存到项目的 `outputs` 文件夹中。你可以在 Finder 中直接查看和访问。

### 09:43 文件的存储与引用

#### 文件存储机制：
- 代理创建的文件保存在项目文件夹的 `outputs` 子目录中
- 聊天记录**不会**保存在文件夹中，只有文件会
- 你可以随时通过 **三点菜单 → Open in Finder** 打开项目文件夹

#### @ 引用文件：
- 在同一项目下的不同聊天中，你可以用 **`@文件名`** 来引用之前创建的文件
- 例如：`@codex-desktop-features.xlsx 请做更多研究，看看我有没有遗漏的新功能`
- 这样 AI 就能读取并理解之前创建的文件内容

> 🔑 **这是非常强大的功能**：不同聊天之间可以通过 @ 引用来共享文件上下文！

### 14:29 技能与插件

很多人容易搞混"插件"和"技能"这两个概念。

#### 核心区别：

| 概念 | 类比 | 说明 |
|------|------|------|
| **Skill（技能）** | 可复用的"菜谱" | 针对特定任务的工作流包 |
| **Plugin（插件）** | 可安装的"模块" | 扩展 Codex 能力的安装单元 |

> 💡 其实两者的界限有点模糊，Riley 个人倾向于把它们都当作"技能"来看待。

#### 不同工具的叫法对比：
- **Codex**：叫"Plugins"和"Skills"
- **Claude**：叫"Customize"标签下的"Skills"和"Connectors"

本质上都是**扩展模型能力**的方式。

#### 举个例子：
- 下载 Codex 后，你不能直接让它"检查邮件"
- 你需要给它安装 **Email 技能**
- 安装后，它就能读取和发送邮件了

### 25:26 引导模式 vs 排队模式

这是 Codex 非常独特的一个功能！

#### 消息排队（Queuing）：
- 在很多 AI 工具中，如果你在 AI 工作时输入新指令，它会**等 AI 做完**再执行
- 这是默认行为

#### 引导模式（Steering）：
- 输入指令后，点击 **"Steer"** 按钮
- 你的指令会**立即插入**到 AI 正在执行的过程中
- 你不需要等 AI 做完就能给出反馈和修正

#### 实际应用：
Riley 在 Paper 设计过程中发现按钮重叠了：
1. 截图问题区域
2. 输入："顺便说一下，这里有重叠，你工作的时候修一下"
3. 点击 **Steer**
4. AI 在当前任务完成后**立即**处理了这个反馈

> 🚀 **Steering 非常适合设计类任务**——你可以在 AI 工作的同时实时调整方向！

### 27:35 用 API 创建自己的技能

这是本视频最实用的部分之一！

#### 什么是 API？
> 对非技术同学来说：API 就是一种用代码访问其他软件信息的方式。就像给 AI 一把"钥匙"，让它能打开其他工具的大门。

#### 创建自定义技能的完整流程：

**第 1 步：找到合适的 API**
- 想想你经常重复做的任务
- 搜索是否有对应的 API 服务

Riley 的例子：他经常从 YouTube 提取字幕做研究，于是搜索"YouTube transcript API"。

**第 2 步：获取 API Key**
- 注册 API 服务（比如 SuperData）
- 获取你的 API Key
- SuperData 每月免费提供 100 次字幕提取

**第 3 步：让 Codex 创建技能**
```
skill creator 请创建一个技能，让我能要求你总结某个 YouTube 频道最新 10 个视频的字幕。
使用 SuperData API。这是我的 API Key：[粘贴你的 Key]
你需要研究如何使用 SuperData，然后创建一个技能。
```

> 💡 **输入 `skill creator`** 会触发 Codex 使用技能创建器，让它更擅长创建技能。

**第 4 步：使用新技能**
- 创建技能后，**必须开启新的聊天**才能使用
- 新聊天中输入技能名称即可使用

## Part 2：多任务实战

|------|------|

| **Skill** | 技能，可复用的工作流包 |
